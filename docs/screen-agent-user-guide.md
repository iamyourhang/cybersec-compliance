# Screen 用户试用说明

## 入口

- 用户入口：`/screen`
- 默认可浏览：全球覆盖地图、国家/地区详情、verified 合规条目、官方源链接、生命周期节点。
- 登录后可使用：右侧“合规 Agent”问答。

## 数据口径

- Screen 默认只展示 `verified` 正式数据。
- `candidate`、`suspicious`、`quarantined` 不进入用户侧地图、问答、导出和 RAG。
- 欧盟等区域组织不是国家，但会作为上级辖区继承到成员国市场。例如查看法国时，会同时看到法国本地制度和 EU CRA、RED、EUCC 等欧盟适用要求。
- 每条信息必须能追溯到官方正文页、官方 PDF/HTML、官方源记录或审核证据备注。

## 用户能做什么

1. 在地图上搜索或点击国家/地区/区域组织。
2. 查看该市场的本地制度、区域/上级辖区适用制度、官方源和近 90 天适用节点。
3. 点击条目查看详情，包括官方链接、发布机构、发布/适用日期、真实性证据和生命周期节点。
4. 登录后向合规 Agent 提问，例如：
   - `美国当前已验证的产品网络安全法规、认证和标准有哪些？`
   - `法国对路由器/交换机有哪些强制或自愿网络安全要求？`
   - `CRA 的报告义务和全面适用日期分别是什么？`
   - `当前条目的官方证据链是什么？`

## Agent 边界

- Agent 只回答网络安全合规问题，包括法律法规、认证、标准、产品合规要求、生效提醒、官方证据和规格要求。
- Agent 不回答天气、旅游、股票、普通网络安全事件、攻击情报等无关问题。
- Agent 不联网补证，不直接写入 verified，不修改审核结论。
- 当 verified 证据不足时，Agent 会返回证据不足并创建补源/审核工单。

## 日期和预警口径

- 生效提醒只基于 `effective_date` 和 `compliance_lifecycle_milestones`。
- 对 CRA 这类多阶段法规，必须展示清楚：
  - 已生效日期
  - 机构通知/指定规则适用日期
  - 漏洞与严重事件报告义务日期
  - 主要义务或全面适用日期
- 用户侧默认看近 90 天节点；飞书日报/周报按 30/90/180/360 天窗口汇总。

## 已知限制

- 规格库只对已验证且可解析的本地文件/切片抽取，不基于联网摘要生成。
- 扫描版 PDF 如果暂未 OCR，仍可保留官方链接和源工件，但不保证规格抽取可用。
- 全球覆盖不是“每国都有产品认证”的假设；对于确实未发现产品级网络安全制度的辖区，应标记为“查无独立产品制度”，并保留研究备注。
